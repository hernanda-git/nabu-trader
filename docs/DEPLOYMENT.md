# Deployment Guide

## Fly.io (Cloud — 24/7)

The system runs 24/7 on Fly.io's Singapore region as a single machine.

### Prerequisites

```bash
# Install flyctl
Windows: Download from https://fly.io/docs/flyctl/install/
         Or use the binary at YOUR_HOME\.fly\bin\flyctl.exe

Linux:  curl -L https://fly.io/install.sh | sh

# Login
flyctl auth login
```

### Cross-Platform Note (Windows)

This project is deployed via the **Windows Fly CLI** binary:
```bash
export PATH="$PATH:/c/Users/it26/.fly/bin"
```

Use `flyctl.exe` (or `flyctl` via the PATH export above). Do **not** shell out through WSL `fly` — the Windows binary owns this app's auth token.

### Application Structure

```
nabu-trader/
├── Dockerfile          # Container definition
├── fly.toml            # Fly.io configuration
├── deploy.sh           # Deploy + version bump script
├── config.yaml         # Trading config (embedded in image)
├── .env                # Local secrets (NOT in image)
└── src/                # Application code (copied into image)
```

### Deploy Steps

#### 1. Set Secrets

```bash
flyctl secrets set \
  TELEGRAM_BOT_TOKEN="..." \
  OPENCODE_GO_API_KEY="sk-..." \
  BINANCE_API_KEY="..." \
  BINANCE_API_SECRET="..." \
  SESSION_STRING="..." \
  API_KEY="$(openssl rand -hex 32)" \
  --app nabu-trader
```

| Secret | Required | Description |
|--------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | ✅ | Bot token from @BotFather |
| `OPENCODE_GO_API_KEY` | ✅ | OpenCode Go API key for LLM |
| `BINANCE_API_KEY` | ⚠️ | Not needed with paper trading or gateway proxy |
| `BINANCE_API_SECRET` | ⚠️ | Not needed with paper trading or gateway proxy |
| `SESSION_STRING` | ✅ | Telethon string session (generated by auth.py) |
| `API_KEY` | ⚠️ | Required for HTTP API bridge |

#### 2. Deploy

```bash
# From project directory
flyctl deploy --app nabu-trader

# Or using the deploy script (bumps version, git tags)
bash deploy.sh
```

That's it — the Dockerfile handles everything:
1. Copies `src/` into the container
2. Installs Python deps from `requirements.txt`
3. Embeds `config.yaml`
4. Starts via `CMD python3 src/main.py`

#### 3. Verify

```bash
flyctl status --app nabu-trader
# Should show: STATE = started, VERSION = latest

flyctl logs --app nabu-trader --no-tail | tail -20
# Should show startup messages, no errors
```

### Container Details

```yaml
# Dockerfile highlights
FROM python:3.11-slim
WORKDIR /app

# Only production dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Source code + config (NOT .env — secrets go to Fly)
COPY src/ src/
COPY config.yaml .

# Persistent data volume
RUN mkdir -p /data/sessions /data/logs

CMD ["python3", "src/main.py"]
```

```yaml
# fly.toml highlights
app = 'nabu-trader'
primary_region = 'sin'

[env]
  CHANNEL_USERNAME = 'YOUR_SIGNAL_CHANNEL'
  DATA_ROOT = '/data'

[http_service]
  internal_port = 9090
  force_https = true

[[mounts]]
  source = 'data_vol'
  destination = '/data'
```

### Persistent Volume

The app uses a Fly volume mounted at `/data/` for persistent state:

```bash
# Check volume status
flyctl volumes list --app nabu-trader

# Data stored:
# /data/trades.db    — SQLite database (all trade history)
# /data/sessions/    — Session backup
# /data/logs/        — Application logs
```

The volume survives deploys, restarts, and machine moves.

### Updating Config

Config is embedded in the Docker image. To change `config.yaml`:

```bash
# Option 1: Edit locally, re-deploy
vim config.yaml
flyctl deploy --app nabu-trader

# Option 2: Use environment variables for runtime changes
# (config keys can be overridden by env vars via config/loader.py)

# Option 3: Via API bridge (requires API_KEY)
curl -X POST -H "X-API-Key: <key>" \
  https://nabu-trader.fly.dev/api/v1/config/update \
  -H "Content-Type: application/json" \
  -d '{"risk": {"port_usdt": 2.0}}'
```

## Local Development

### Prerequisites

```bash
cd nabu-trader
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your keys
```

### Running Locally

```bash
# Dry-run (no real trades)
# Edit config.yaml: agent.auto_trade: false
python src/main.py

# Paper trading
# Edit config.yaml: exchange.active: paper
python src/main.py

# Real trading (testnet)
# Edit config.yaml: exchange.active: binance_testnet
python src/main.py

# Real trading (mainnet)
# Edit config.yaml: exchange.active: binance
python src/main.py
```

### Run Tests

```bash
python -m pytest tests/ -v
```

## Git Workflow

```bash
# Branch for features
git checkout -b feature/my-change

# Make changes, then:
git add src/ docs/
git commit -m "description: what changed"
git push origin feature/my-change

# Deploy from any branch
flyctl deploy --app nabu-trader

# Merge to main
git checkout main
git merge feature/my-change
git push origin main
```

The `deploy.sh` script automates version bump, commit, and deploy:

```bash
bash deploy.sh
```

This:
1. Reads current version from `src/version.py`
2. Bumps version (e.g. v101 → v102)
3. Commits with version tag
4. Deploys to Fly.io
5. Verifies the new version is running

## Environment Matrix

| Mode | `exchange.active` | `agent.auto_trade` | API Calls | Risk |
|------|-------------------|-------------------|-----------|------|
| Dry-run | any | `false` | Read-only | None |
| Paper trading | `paper` | `true` | None | None |
| Testnet | `binance_testnet` | `true` | Binance testnet | Fake money |
| Production | `binance` | `true` | Real Binance | Real money |
| Gateway proxy | `binance` + proxy | `true` | Via gateway | Real money |
